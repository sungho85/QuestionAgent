# System Management Web Tool Sample v1 (KO)

## 개요
이 문서는 Web Master Plan Template v1이 실제로 어떻게 채워질 수 있는지를 보여주는
예시용 샘플 시나리오이다.

본 샘플은 **비개발자가 사내에서 사용하는 시스템 관리용 웹 도구**를 가정한다.

---

## 1. 프로젝트 개요

### 목적
사내에서 실행되는 작업 요청과 처리 상태를 관리자가 확인하고 관리할 수 있는 내부 관리 도구를 제공한다.

### 대상 사용자
- 관리자
- 내부 직원

### 사용 환경
- 실행 환경: 사내 서버
- 인터넷 연결: 필요

---

## 2. 기능 범위

### 로그인 및 인증
- 로그인 기능 사용
- 관리자 전용 기능 존재

### 핵심 관리 대상 (CRUD)
- 관리 대상 이름: 작업 요청
- 설명: 내부에서 발생하는 작업 실행 요청을 관리
- 기능:
  - 목록 조회
  - 상세 보기
  - 신규 등록
  - 수정
  - 삭제

### 시스템 설정
- 시스템 설정 관리 기능 사용
- 예시 설정:
  - 최대 동시 작업 수
  - 기본 작업 우선순위

### 로그 관리
- 로그 관리 기능 사용
- 로그 종류:
  - 접속 기록
  - 작업 실행 기록
  - 상태 변경 이력

---

## 3. 화면 구성

- 로그인 화면
- 대시보드 화면
- 작업 요청 리스트 화면
- 작업 요청 상세 화면
- 작업 요청 수정 화면
- 시스템 설정 화면
- 로그 조회 화면

---

## 4. 데이터 구조

### 사용자
- 아이디
- 이름
- 역할
- 상태

### 작업 요청
- 요청 ID
- 요청 제목
- 요청 내용
- 상태
- 생성 시간
- 수정 시간

### 로그
- 로그 ID
- 발생 시간
- 사용자
- 이벤트 내용

---

## 5. 운영 정책

- 기본 관리자 계정 자동 생성
- 관리자만 설정 및 로그 접근 가능
- 로그 보관 기간: 90일

---

## 6. 개발 실행 계획 (요약)

1. 프로젝트 생성
2. 기본 프레임워크 설정
3. 데이터베이스 초기화
4. 인증 및 권한 기능 구현
5. 작업 요청 CRUD 구현
6. 관리용 UI 템플릿 적용
7. 로그 및 설정 기능 구현
8. 테스트 및 검증
9. 결과 리포트 생성

---

## 비고
본 문서는 템플릿 검증 및 QuestionAgent 동작 확인을 위한 예시이다.
