# QuestionAgent 웹 프로토타입 목표 정의

## 1. 프로토타입의 목표

이번 QuestionAgent 프로토타입의 목표는 다음과 같다.

비개발자가 채팅만으로 간단한 사내 웹툴 수준의 웹 애플리케이션을 만들 수 있도록  
**개발 명세서 및 마스터 플랜을 자동 생성**하는 기능을 구축한다.

본 프로토타입은 단순한 1~2페이지짜리 정적 웹사이트가 아니라  
프레임워크, 로컬 DB, 로그인 기능, CRUD 기능이 포함된  
**간단한 업무용 웹 애플리케이션**을 생성할 수 있어야 한다.

---

## 2. 결과물 요약

프로토타입이 생성해야 할 결과물:

1. 프레임워크 기반 프로젝트 구조  
2. 로그인 페이지  
3. 리스트 페이지  
4. 상세 정보 페이지  
5. 등록/수정/삭제 페이지  
6. 템플릿(부트스트랩 등)이 적용된 UI  
7. 로컬 DB(SQLite 등) 기반 데이터 저장

유저는 기술 지식 없이 자연어 설명만 제공하며  
Codex가 마스터 플랜에 따라 자동으로 코드를 생성하도록 한다.

---

## 3. 필요한 기능

### 3.1 사용자 친화적 질문 구조
비개발자는 기술 용어를 이해하지 못하므로 다음과 같은 질문은 부적절하다.

- DB 무엇 사용?  
- 프레임워크 무엇 선택?  
- 템플릿 무엇 적용?

대신 목적 중심 질문을 사용한다.

예시:
- “이 웹툴을 어디서 사용할 예정인가요?”  
- “몇 명 정도 사용할까요?”  
- “로그인은 반드시 필요할까요?”  
- “데이터를 얼마나 오래 보관해야 할까요?”

---

### 3.2 LLM 기반 질문 생성기

내부적으로 필요한 기술 스펙(DB, 프레임워크, 인증 방식 등)을  
사용자 친화적인 질문 세트로 변환해야 한다.

구조는 다음과 같다.

1) 내부 기술 요구 목록  
2) 사용자 친화적 질문 목록  
3) 사용자의 답변을 기술 스펙으로 매핑하는 변환 로직  
4) 매핑된 기술 스펙을 기반으로  
   - DB 구조  
   - 라우팅  
   - 페이지 목록  
   - 템플릿 구조  
   - 코드 생성 프롬프트  
   를 자동 구성

---

## 4. 마스터 플랜 동작 요구 사항

Codex에 마스터 플랜을 넣으면 다음이 자동으로 실행되어야 한다.

1. 프로젝트 생성  
2. 프레임워크 설치 및 초기 설정  
3. 스텝 단위 플랜 생성  
4. 프롬프트 자동 생성  
5. 코드 생성 (한/영 병기 주석 포함)  
6. 단계별 코드 검증  
7. 결과 리포트 저장  

사용자는 채팅만 입력하거나 실행 로그를 확인하면 되고  
모든 개발 작업은 Codex가 수행한다.

---

## 5. 제공해야 하는 가이드

### 5.1 환경 설치 가이드
- VS Code 설치 방법  
- Codex 또는 AntiGravity 설치 방법  
- 확장 기능, 실행 방법

### 5.2 디버깅 가이드
- 에러 발생 시 메시지를 복사하여 Codex에 문의하는 방법  
- 웹 프로젝트의 구조와 기본 실행 방식 설명

---

## 6. 핵심 목표 요약

- 비개발자도 CRUD 웹툴을 만들 수 있어야 함  
- 기술 선택(DB, 프레임워크 등)은 사용자에게 직접 묻지 않음  
- LLM이 질문을 생성하고 기술 스펙으로 매핑  
- Codex가 프로젝트 생성 → 코드 생성 → 검증까지 수행  
- QuestionAgent는 전체 플로우의 설계·명세·질문 담당
