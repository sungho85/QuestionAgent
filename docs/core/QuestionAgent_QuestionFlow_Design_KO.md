# QuestionAgent 질문 기반 마스터 플랜 생성 구조 (KO)

## 1. 개요
이 문서는 QuestionAgent가 비개발자를 대상으로 웹/마야/콘솔 툴 등 어떤 도메인이든 적용할 수 있는 **질문 기반 마스터 플랜 생성 방식**을 정의한 것이다.

## 2. 핵심 개념
QuestionAgent는 사용자의 수준과 요구사항을 기반으로 LLM을 활용하여:
1. 사용자 수준 파악  
2. 도메인별 마스터 플랜 템플릿 + 사용자 요구를 LLM에 전달  
3. LLM이 사용자 수준에 적합한 전체 질문 리스트 생성  
4. 질문 하나씩 진행하며 답변을 템플릿의 빈 항목에 채움  
5. 채워진 정보를 다시 LLM에 전달하여 마스터 플랜을 자동 완성  
6. 완성된 마스터 플랜을 Codex 등 코드 생성 엔진이 실행할 수 있도록 전달

이 과정을 통해 비개발자도 채팅만으로 프로그램 설계를 완성할 수 있다.

## 3. 전체 흐름

### 1단계: 사용자 초기 입력
- 만들고 싶은 것에 대한 자연어 설명  
- 사용자 수준(비개발자 / 초급자 / PM / 개발자)

### 2단계: LLM 기반 질문 리스트 생성
마스터 플랜 템플릿 + 유저 초기 설명을 LLM에 보내  
사용자 수준에 최적화된 질문 리스트(질문 골격)을 생성한다.

### 3단계: 질문-답변 기반 설계 완성
LLM이 생성한 질문을 하나씩 제시하고  
답변을 해당 마스터 플랜 항목에 자동으로 채워 넣는다.

### 4단계: 보완 및 최종 마스터 플랜 생성
부분적으로 채워진 마스터 플랜 + 답변 전체를 다시 LLM에 보내  
보완·정리·누락 검증을 수행해 최종 형태로 완성한다.

### 5단계: Codex 실행 연결
완성된 마스터 플랜은 Codex가 다음을 수행할 수 있는 구조를 갖춘다.
- 프로젝트 생성  
- 코드 생성  
- 기능 검증  
- 결과 보고서 생성  

## 4. 구조적 장점
- 도메인에 독립적인 구조  
- 사용자 수준에 맞춘 질문 난이도 조절  
- 템플릿 기반 확장성 (웹, 마야, UE5 등 추가 가능)  
- LLM이 질문을 생성해 UX 장벽 감소  
- 마스터 플랜 자동 검증 가능  

## 5. 요약
이 구조는 QuestionAgent가 비개발자부터 개발자까지 누구나  
자연어 기반으로 프로그램 설계가 가능하도록 만드는 핵심 메커니즘이다.
