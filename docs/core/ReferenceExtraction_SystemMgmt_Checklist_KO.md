# Reference Extraction Checklist
## 시스템 관리 페이지 기준 체크리스트 (KO)

이 문서는 실제 존재하는 시스템 관리용 웹 프로젝트를 참고하여,
Web Master Plan Template에 반드시 반영되어야 할 항목을 빠짐없이 추출하기 위한 체크리스트이다.

본 문서는 구현 방법이나 코드와 무관하며, 구조와 기능 존재 여부만 점검하는 것을 목적으로 한다.

---

## 1. 인증 및 권한

- [ ] 로그인 기능 존재
- [ ] 관리자 / 일반 사용자 구분
- [ ] 관리자 전용 접근 화면 존재
- [ ] 권한에 따라 메뉴 또는 기능 제한

---

## 2. 핵심 관리 대상 (CRUD)

- [ ] 관리 대상 엔터티가 명확히 정의되어 있음
- [ ] 목록(List) 화면 존재
- [ ] 상세(Detail) 화면 존재
- [ ] 등록(Create) 기능 존재
- [ ] 수정(Update) 기능 존재
- [ ] 삭제(Delete) 기능 존재

---

## 3. 시스템 설정

- [ ] 시스템 설정 데이터 존재
- [ ] 설정 키 / 설명 / 값 구조
- [ ] 관리자만 설정 변경 가능
- [ ] 설정 변경 이력 관리 여부

---

## 4. 로그 및 기록

- [ ] 사용자 접속 로그
- [ ] 작업 또는 실행 로그
- [ ] 데이터 변경 이력
- [ ] 관리자 조회 화면 제공

---

## 5. UI 구조

- [ ] 상단 바 또는 헤더 영역
- [ ] 사이드 메뉴 또는 네비게이션
- [ ] 테이블 기반 목록 UI
- [ ] 공통 레이아웃 적용

---

## 6. 운영 관점

- [ ] 초기 관리자 계정 생성 방식 정의
- [ ] 시스템 사용 범위(사내/로컬) 명확
- [ ] 데이터 보관 정책 고려

---

## 사용 방법

1. 참고 프로젝트를 기능 단위로 살펴본다.
2. 위 체크리스트 항목을 기준으로 존재 여부만 표시한다.
3. 누락된 항목은 Web Master Plan Template에 반영한다.
