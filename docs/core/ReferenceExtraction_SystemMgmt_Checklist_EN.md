# Reference Extraction Checklist
## System Management Page Checklist (EN)

This document is a checklist used to extract essential structural elements
from existing system management web projects.

It focuses only on feature presence and structure, not implementation details or code.

---

## 1. Authentication and Authorization

- [ ] Login functionality exists
- [ ] Admin vs regular user roles
- [ ] Admin-only pages or features
- [ ] Permission-based access control

---

## 2. Core Managed Entities (CRUD)

- [ ] Clearly defined managed entity
- [ ] List view
- [ ] Detail view
- [ ] Create functionality
- [ ] Update functionality
- [ ] Delete functionality

---

## 3. System Configuration

- [ ] System configuration data exists
- [ ] Key / description / value structure
- [ ] Admin-only configuration changes
- [ ] Configuration change history

---

## 4. Logs and Records

- [ ] User access logs
- [ ] Task or execution logs
- [ ] Data change history
- [ ] Admin log viewing interface

---

## 5. UI Structure

- [ ] Top bar or header
- [ ] Side navigation menu
- [ ] Table-based list UI
- [ ] Shared layout across pages

---

## 6. Operational Perspective

- [ ] Default admin account strategy
- [ ] Usage scope defined (local / internal)
- [ ] Data retention policy considered

---

## How to Use

1. Review a reference system management project by features.
2. Mark existing items using this checklist.
3. Reflect missing items into the Web Master Plan Template.
